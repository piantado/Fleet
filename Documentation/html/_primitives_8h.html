Detailed Description

A preprimitive is a class that primitives inherit. We use a static op_counter so that at compile time thsi can be used to compute an op number for each separate primitive.

A primitive associates a string name (format) with a function, and allows grammars to extract all the relevant function pieces via constexpr, and also defines a VMS function that can be called in dispatch op here is generated uniquely for each Primitive, which is how LOTHypothesis knows which to call. SO: grammars use this op (statically updated) in nodes, and then when they are linearized the op is processed in a switch statement to evaluate This is a funny intermediate type that essentially helps us associate functions, formats, and nonterminal types all together.
